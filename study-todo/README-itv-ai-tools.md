## 1. Claude Code

### 1.1 基本使用

#### 安装流程

```bash
# 通过VS Code扩展市场安装
# 1. 打开VS Code
# 2. 点击扩展图标（Ctrl+Shift+X）
# 3. 搜索 "Claude Code"
# 4. 点击安装
```

#### 界面导航

| 区域 | 功能说明 | 快捷键 |
|------|---------|--------|
| 侧边栏 | 模型选择、历史记录 | `Ctrl+Shift+P` → "Claude" |
| 聊天面板 | AI对话界面 | `Ctrl+Shift+C` |
| 代码编辑器 | 代码编辑与提示 | 常规编辑快捷键 |

#### 基础功能操作

```python
# 1. 代码补全
# 在编辑器中输入代码，Claude会自动提供补全建议

# 2. 代码解释
# 选中代码 → 右键 → "Explain with Claude"

# 3. 代码生成
# 使用命令面板 → "Generate Code with Claude"
```

#### 入门级使用场景

- **快速生成代码**：根据自然语言描述生成代码片段
- **代码解释**：理解复杂代码的功能和逻辑
- **代码优化建议**：获取代码改进建议
- **文档生成**：为代码生成注释和文档

### 1.2 高阶用法

#### 高级功能

```python
# 多文件上下文理解
# Claude可以理解项目中的多个文件关联关系

# 代码重构建议
# 选中代码块 → "Refactor with Claude"

# 测试用例生成
# 在函数上方添加注释 → Claude自动生成测试
```

#### 复杂任务处理

```python
# 完整项目生成
prompt = """
请帮我创建一个完整的待办事项应用，包含：
1. React前端
2. Node.js后端
3. MongoDB数据库
4. RESTful API

要求：
- 使用TypeScript
- 包含用户认证
- 支持任务分类和优先级
"""
# 将此prompt输入Claude，获取完整项目代码
```

#### 效率提升技巧

1. **自定义指令**：创建个性化的prompt模板
2. **上下文管理**：利用长上下文能力处理大型代码库
3. **批量处理**：一次性处理多个相关任务

### 1.4 Skill技能

#### 技能体系

| 技能类型 | 描述 | 示例 |
|---------|------|------|
| 代码生成 | 根据需求生成代码 | 生成排序算法 |
| 代码分析 | 分析代码结构和逻辑 | 性能分析 |
| 代码重构 | 优化代码质量 | 重构设计模式 |
| 文档生成 | 生成技术文档 | API文档 |
| 测试生成 | 生成测试用例 | 单元测试 |

#### 自定义技能创建

```python
# 创建自定义技能
custom_skill = {
    "name": "代码安全检查",
    "description": "检查代码中的安全漏洞",
    "prompt": """
    请检查以下代码中的安全漏洞：
    
    {code}
    
    需要检查：
    1. SQL注入风险
    2. XSS漏洞
    3. 敏感数据泄露
    4. 输入验证不足
    
    请给出修复建议。
    """
}
```

#### 技能组合策略

```python
# 技能组合示例
# 1. 使用"需求分析"技能理解需求
# 2. 使用"架构设计"技能设计方案

# 4. 使用"代码审查"技能检查质量
# 5. 使用"测试生成"技能编写测试
```

### 1.5 使用规范

#### 标准化操作流程

```
1. 明确需求 → 2. 编写清晰prompt → 3. 生成代码 → 4. 审查验证 → 5. 测试确认
```

#### 命名约定

```python
# Prompt命名规范
# 格式：[任务类型]-[描述].prompt
# 示例：
# - code-generate-user-auth.prompt
# - docs-generate-api.prompt
# - review-security-check.prompt
```

#### 输出格式要求

```python
# 代码输出格式要求
# 1. 包含必要的注释
# 2. 遵循项目代码风格
# 3. 包含错误处理
# 4. 提供使用示例
```

#### 安全使用准则

- 🔒 不向AI透露敏感信息（API密钥、密码等）
- 🔒 不处理涉密代码
- 🔒 对生成代码进行安全审查
- 🔒 使用企业级部署时启用数据隔离

### 1.6 高级特性与技巧

#### 隐藏功能

```python
# 提示工程进阶技巧
# 使用XML标签结构化prompt
prompt = """
<task>代码优化</task>
<input>
{code}
</input>
<requirements>
1. 提高性能
2. 增强可读性
3. 添加错误处理
</requirements>
<output_format>
优化后的代码 + 变更说明
</output_format>
"""
```

#### 性能优化方法

| 方法 | 说明 | 效果 |
|------|------|------|
| 精简prompt | 删除冗余内容 | 减少Token消耗 |
| 使用模板 | 复用标准化prompt | 提高一致性 |
| 分段处理 | 大型任务拆分为多个小任务 | 提高成功率 |

#### 问题排查技巧

```python
# 常见问题及解决方法

# 问题1：生成代码不符合预期
# 解决：提供更详细的需求描述和示例

# 问题2：上下文丢失
# 解决：使用文件引用或摘要

# 问题3：响应时间过长
# 解决：缩短单次请求内容，使用分批处理
```

## 3. Cursor

### 3.1 功能概述

Cursor是一款专为AI辅助编程设计的代码编辑器，基于VS Code构建，集成了强大的AI功能。

### 3.2 核心特点

| 特点 | 描述 |
|------|------|
| AI优先 | 深度集成AI能力 |
| 智能补全 | 上下文感知的代码补全 |
| 聊天界面 | 内置AI聊天窗口 |
| 代码理解 | 强大的代码分析能力 |

### 3.3 核心使用要点

```python
# 快捷键
# Cmd+K (Mac) / Ctrl+K (Win) - 打开AI聊天
# Cmd+L (Mac) / Ctrl+L - 选择代码并询问

# 基本操作
# 1. 输入代码时自动补全
# 2. 选中代码后按Cmd+L询问
# 3. 使用聊天窗口进行自然语言交互
```

### 3.4 配置要点

```json
// settings.json
{
  "cursor.model": "gpt-4",
  "cursor.autoComplete": true,
  "cursor.chatPosition": "right"
}
```
