# 家族树项目功能规划文档

## 项目概述
本项目是一个基于MCP协议的家族树管理应用，提供家族管理、成员管理、关系管理、家族树可视化等核心功能。

## 现有功能清单

| 模块 | 功能 | 状态 |
|------|------|------|
| 用户认证 | 登录、注册、个人中心、修改密码 | ✅ 完整 |
| 家族管理 | 创建/编辑/删除家族、邀请成员、权限管理 | ✅ 完整 |
| 成员管理 | 添加/编辑/删除成员、搜索成员 | ✅ 完整 |
| 关系管理 | 创建/删除关系、关系称谓计算 | ✅ 完整 |
| 家族树 | 可视化展示家族成员关系图 | ✅ 完整 |
| 统计分析 | 成员数量、性别比例、年龄分布 | ✅ 完整 |
| 历史记录 | 事件创建/删除、时间线展示 | ✅ 完整 |
| 家族相册 | 相册管理、照片上传/删除 | ✅ 完整 |
| 家族动态 | 发布动态、点赞、评论 | ✅ 完整 |
| 纪念堂 | 逝者纪念、墓志铭 | ✅ 完整 |

## 功能规划路线图

### 第一阶段：核心功能补全

#### 1. 成员详情页
- **优先级**：🔴 高
- **状态**：✅ 已完成
- **功能描述**：点击成员时，展示完整的成员档案页面
- **包含内容**：
  - 基本信息卡片（姓名、性别、出生日期、联系方式）
  - 关系图谱（显示与其他成员的关系网络）
  - 相关照片（关联该成员的相册照片）
  - 生平事件时间线（该成员相关的历史事件）
- **实现文件**：`src/server/ui/memberDetailHtml.ts` + 对应工具函数

#### 2. 数据持久化
- **优先级**：🔴 高
- **状态**：✅ 已完成
- **问题**：所有Store都是内存存储，服务器重启后数据全部丢失
- **解决方案**：引入JSON文件存储
- **实现方案**：
  - 创建数据目录 `data/`
  - 实现定期保存机制（每30秒或操作后）
  - 实现启动时自动加载机制
- **集成范围**：familyStore、memberStore、relationshipStore、historyStore、albumStore、feedStore、memorialStore、eventStore、permissionStore

### 第二阶段：体验增强

#### 3. 生日/纪念日提醒
- **优先级**：🟡 中
- **状态**：✅ 已完成
- **功能描述**：在首页展示即将到来的生日和重要纪念日
- **包含内容**：
  - 近期生日提醒卡片（未来14天内过生日的成员）
  - 纪念日提醒（里程碑事件、重要事件等）
  - 统计本月有多少人生日和纪念日
- **实现文件**：`src/server/tools/familyTools.ts` (getUpcomingRemindersTool) + `src/server/ui/homeHtml.ts`

#### 4. 全局搜索
- **优先级**：🟡 中
- **状态**：✅ 已完成
- **功能描述**：统一搜索入口，支持搜索成员、家族、相册、动态、纪念堂
- **包含内容**：
  - 搜索框支持按类型筛选（成员/家族/相册/动态/纪念堂）
  - 搜索结果按类型分组展示
  - 支持点击结果跳转对应页面
- **实现文件**：`src/server/tools/familyTools.ts` (globalSearchTool + getSearchUITool) + `src/server/ui/searchHtml.ts`

### 第三阶段：完善与优化

#### 5. 关系校验
- **优先级**：🟡 中
- **状态**：✅ 已完成
- **问题**：创建关系时缺少循环依赖和一致性校验
- **解决方案**：
  - 校验不能创建自关系
  - 校验两个成员之间不能重复创建关系
  - 校验配偶关系必须是不同性别，男性创建丈夫关系，女性创建妻子关系
  - 校验父母关系必须匹配性别（男性创建父亲，女性创建母亲）
  - 校验子女关系必须匹配性别（男性不能是女儿，女性不能是儿子）
  - 校验不能同时创建父子/母子和子父/子母关系（防止循环依赖）
- **实现文件**：`src/server/relationshipStore.ts` (validateRelationship方法) + `src/server/tools/familyTools.ts` (createRelationshipTool)

#### 6. 数据导出
- **优先级**：🟢 低
- **状态**：✅ 已完成
- **功能描述**：支持将家族数据导出为JSON格式
- **包含内容**：
  - 支持按类型导出（成员/家族/关系/事件/相册/动态/纪念堂）
  - 支持按家族筛选导出
  - 支持导出全部数据（包含用户和权限）
- **实现文件**：`src/server/tools/familyTools.ts` (exportDataTool)

#### 7. 安全加固
- **优先级**：🟢 低
- **状态**：✅ 已完成
- **问题**：密码明文存储，无Session过期机制
- **解决方案**：
  - 使用bcrypt加密密码（10轮盐值）
  - 添加Session过期机制（24小时自动过期，每次访问刷新）
- **实现文件**：`src/server/familyStore.ts` (bcrypt集成) + `src/server/tools/familyTools.ts` (Session管理)

#### 8. 通知系统
- **优先级**：🟢 低
- **状态**：待开发
- **功能描述**：实时通知重要事件
- **包含内容**：
  - 邀请加入家族通知
  - 动态点赞/评论通知
  - 重要事件提醒

## 功能流程图

### 成员详情页流程
```
用户点击家族树/成员列表中的成员
    ↓
调用 getMemberDetailUI 工具
    ↓
生成 memberDetailHtml 页面
    ↓
页面加载时调用：
  ├─ getMemberById → 获取成员基本信息
  ├─ listRelationships → 获取该成员的所有关系
  ├─ listPhotos → 获取关联照片
  └─ listEvents → 获取相关事件
    ↓
渲染详情页面
    ↓
用户可执行操作：
  ├─ 编辑成员信息
  ├─ 添加关系
  ├─ 查看关系网络
  └─ 返回列表
```

### 数据持久化流程
```
服务器启动
    ↓
检查 data/ 目录是否存在
    ↓
加载各Store数据（从JSON文件）
    ↓
正常运行，处理用户请求
    ↓
操作后触发保存（或定期保存）
    ↓
将各Store数据写入JSON文件
    ↓
服务器关闭时最后保存一次
```
