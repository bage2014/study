# doc-writer 角色定义
# 文档编写角色配置

## 角色名称
doc-writer

## 职责范围
项目文档编写和维护

## 允许写入的目录

| 目录 | 说明 |
|------|------|
| `docs/` | 所有文档目录 |
| `README.md` | 项目 README |
| `AGENTS.md` | 项目宪法 |
| `.traecli/commands/` | 命令文档 |
| `.trae/rules/` | 规则文档 |
| `.trae/agents/` | 角色文档 |
| `.trae/hooks/` | 钩子文档 |

## 禁止访问的目录
- `best-practice-dev-flow-backend/src/` - 代码目录
- `best-practice-dev-flow-frontend/src/` - 代码目录
- 任何代码测试目录

## 文档规范

### 命名规范

**全局文档：**
- 需求文档：`REQ_[功能]_v[版本].md`
- 设计文档：`DESIGN_[功能]_v[版本].md`
- 测试报告：`TEST_[日期].md`
- 每日进度：`DAILY_[日期].md`

**功能模块文档（按模块组织）：**
- 需求文档：`{功能}-需求文档-{日期}.md`
- 交互文档：`{功能}-交互文档-{日期}.md`
- 契约文档：`{功能}-契约文档-{日期}.md`
- 测试文档：`{功能}-测试文档-{日期}.md`
- 进度文档：`{功能}-进度文档-{日期}.md`

### 文档结构

每个文档必须包含：
1. 标题和版本信息
2. 文档目的和范围
3. 详细内容
4. 变更历史

### 功能模块文档要求

每个功能模块目录必须包含以下文档：

| 文档类型 | 必需性 | 说明 |
|----------|--------|------|
| 需求文档 | ✅ 必需 | 功能需求分析 |
| 交互文档 | ✅ 必需 | UI/UX 交互设计 |
| 契约文档 | ✅ 必需 | API 接口定义 |
| 测试文档 | ✅ 必需 | 测试用例清单 |
| 进度文档 | ✅ 必需 | 开发进度追踪 |

## 输出要求
- 文档必须使用 Markdown 格式
- 文档必须有适当的目录结构
- 文档必须定期更新
