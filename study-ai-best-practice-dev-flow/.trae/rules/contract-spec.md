# 契约生成规范文档

## 一、文档结构

```
docs/features/{功能}/
└── {功能}-契约文档-{日期}.md
```

## 二、内容模板

### 1. API 列表

| API 路径 | HTTP 方法 | 功能描述 |
|----------|----------|----------|
| `/api/users/login` | POST | 用户登录 |
| `/api/users` | GET | 获取用户列表 |

### 2. 请求结构

**POST /api/users/login**

请求体：
```json
{
  "username": "string (必填，用户名)",
  "password": "string (必填，密码)"
}
```

### 3. 响应结构

成功响应 (200)：
```json
{
  "id": "number (用户ID)",
  "username": "string (用户名)",
  "email": "string (邮箱)",
  "token": "string (JWT令牌)"
}
```

错误响应 (401)：
```json
{
  "message": "string (错误信息)"
}
```

### 4. 状态码

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 500 | 服务器错误 |

### 5. 字段说明

| 字段 | 类型 | 含义 | 约束 |
|------|------|------|------|
| id | Long | 用户ID | 非空 |
| username | String | 用户名 | 长度3-20 |
| email | String | 邮箱 | 符合邮箱格式 |

## 三、同步机制

- API 变更必须更新契约文档
- 前后端根据契约文档并行开发
- 接口联调前验证契约一致性

## 四、命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 文件 | `docs/features/{功能}/{功能}-契约文档-{日期}.md` | `docs/features/login/登录-契约文档-2026-05-12.md` |
