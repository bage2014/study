# 文档规范

## 一、文档类型

### 1.1 需求文档（PRD）

**文件路径**：`docs/features/{功能}/{功能}-需求文档-{日期}.md`

**内容结构**：
| 章节 | 说明 |
|------|------|
| 需求概述 | 需求来源、功能描述、业务价值 |
| 功能需求 | 需求点列表、优先级 |
| 非功能需求 | 性能、安全、兼容性要求 |
| 数据流向 | 数据流转说明 |
| 验收标准 | 功能验收条件 |

### 1.2 交互设计文档

**文件路径**：`docs/features/{功能}/{功能}-交互设计文档-{日期}.md`

**内容结构**：
| 章节 | 说明 |
|------|------|
| 页面结构 | 页面布局、组件划分 |
| 交互流程 | 操作流程、状态转换 |
| 界面原型 | 页面设计稿、交互说明 |
| 状态说明 | 各种状态下的界面表现 |

### 1.3 技术设计文档

**文件路径**：`docs/features/{功能}/{功能}-技术设计文档-{日期}.md`

**内容结构**：
| 章节 | 说明 |
|------|------|
| 架构设计 | 技术选型、架构图 |
| 数据库设计 | 表结构、关系图 |
| API设计 | 接口定义、参数说明 |
| 类设计 | 核心类结构 |

### 1.4 测试文档

**文件路径**：`docs/features/{功能}/{功能}-测试文档-{日期}.md`

**内容结构**：
| 章节 | 说明 |
|------|------|
| 测试概述 | 测试范围、测试目标 |
| 测试用例 | 单元测试、接口测试、UI测试 |
| 测试数据 | 测试数据说明 |
| 缺陷记录 | 发现的问题及处理 |

### 1.5 接口契约文档

**文件路径**：`docs/features/{功能}/{功能}-契约文档-{日期}.md`

**内容结构**：
| 章节 | 说明 |
|------|------|
| API列表 | 接口路径、方法、功能描述 |
| 请求结构 | 请求参数、数据类型 |
| 响应结构 | 响应字段、数据类型 |
| 状态码 | 返回状态码说明 |

## 二、文档命名规范

| 文档类型 | 文件名格式 | 示例 |
|----------|----------|------|
| 需求文档 | `{功能}-需求文档-{日期}.md` | `登录-需求文档-2026-05-12.md` |
| 交互设计 | `{功能}-交互设计文档-{日期}.md` | `登录-交互设计文档-2026-05-12.md` |
| 技术设计 | `{功能}-技术设计文档-{日期}.md` | `登录-技术设计文档-2026-05-12.md` |
| 测试文档 | `{功能}-测试文档-{日期}.md` | `登录-测试文档-2026-05-12.md` |
| 契约文档 | `{功能}-契约文档-{日期}.md` | `登录-契约文档-2026-05-12.md` |

## 三、文档版本管理

### 3.1 版本号规则

```
V{大版本}.{小版本}.{修订号}
```

示例：
- V1.0.0 - 初始版本
- V1.1.0 - 功能增强
- V1.1.1 - Bug修复

### 3.2 更新记录

每个文档必须包含更新记录章节：

```markdown
## 更新记录

| 版本 | 日期 | 修改人 | 修改内容 |
|------|------|--------|----------|
| V1.0.0 | 2026-05-12 | 张三 | 初始版本 |
| V1.1.0 | 2026-05-15 | 李四 | 新增XX功能 |
```

## 四、文档编写规范

### 4.1 语言规范

- 使用中文编写
- 语言简洁、清晰
- 避免使用口语化表达

### 4.2 格式规范

- 使用 Markdown 格式
- 标题层级清晰（#、##、###）
- 使用表格展示结构化数据
- 使用代码块展示代码示例

### 4.3 内容规范

- 需求描述明确、可验证
- 界面描述包含布局、交互、状态
- 技术设计包含架构、流程、接口定义
- 测试文档包含测试用例、预期结果

## 五、文档同步机制

### 5.1 更新触发条件

当以下情况发生时，必须同步更新相关文档：

| 触发事件 | 需要更新的文档 |
|----------|---------------|
| 需求变更 | PRD需求文档、交互设计文档 |
| 设计变更 | 交互设计文档、技术设计文档 |
| 接口变更 | 接口契约文档 |
| 功能上线 | 测试文档 |

### 5.2 审批流程

1. 文档编写完成
2. 相关人员审核
3. 确认无误后归档
4. 更新版本号和更新记录

## 六、文档归档

### 6.1 归档位置

所有文档存放在 `docs/features/{功能}/` 目录下。

### 6.2 归档时间

功能上线后一周内完成文档归档。
