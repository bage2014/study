## 一、需求概述

### 1.1 需求来源
- 产品需求：记录家族重要历史事件
- 用户反馈：需要记录成员个人大事件

### 1.2 功能描述
实现家族历史事件和成员大事件的记录功能，支持富文本编辑。

### 1.3 业务价值
- 记录家族历史传承
- 保存成员人生重要时刻
- 方便后代了解家族历史

## 二、功能需求

| 序号 | 需求点 | 描述 | 优先级 |
|------|--------|------|--------|
| REQ_EVENT_001 | 创建事件 | 创建家族重要事件 | 高 |
| REQ_EVENT_002 | 编辑事件 | 修改事件信息 | 高 |
| REQ_EVENT_003 | 删除事件 | 删除事件 | 高 |
| REQ_EVENT_004 | 时间轴展示 | 按时间顺序展示事件 | 高 |
| REQ_EVENT_005 | 成员大事件 | 记录成员个人大事件 | 高 |
| REQ_EVENT_006 | 富文本编辑 | 支持图片和文本编辑 | 高 |

## 三、非功能需求

### 3.1 性能要求
- 事件列表加载时间 ≤ 300ms

### 3.2 安全要求
- 只有管理员可以删除事件

## 四、数据流向

```
创建事件 → 保存到数据库 → 更新时间轴展示
```

## 五、验收标准

- [ ] 创建事件成功
- [ ] 时间轴正确展示事件
- [ ] 成员大事件记录成功
- [ ] 富文本编辑支持图片

## 六、接口定义

### 6.1 创建事件

**POST /api/events**

请求体：
```json
{
  "title": "string (必填，事件标题)",
  "description": "string (可选，描述)",
  "eventDate": "string (可选，事件日期)",
  "location": "string (可选，地点)",
  "familyId": "number (必填，家族ID)"
}
```

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "title": "家族聚会",
    "eventDate": "2026-01-01"
  }
}
```
