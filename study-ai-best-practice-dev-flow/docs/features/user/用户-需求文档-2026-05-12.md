# 用户-需求文档

## 1. 需求概述

### 1.1 功能描述

用户管理功能，允许管理员查看、添加、编辑和删除用户信息。

### 1.2 业务背景

系统需要对用户进行统一管理，包括查看用户列表、创建新用户、修改用户信息和删除用户。

### 1.3 需求来源

用户需求："登陆后，可以查看当前查看用户列表，实现用户的增删改差功能"

---

## 2. 功能需求

### 2.1 核心功能

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| FR-008 | 用户列表展示 | 高 |
| FR-008-01 | 添加新用户 | 高 |
| FR-008-02 | 编辑用户信息 | 高 |
| FR-008-03 | 删除用户 | 高 |

### 2.2 非功能需求

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| NFR-005 | 列表分页显示 | 中 |
| NFR-006 | 删除前确认提示 | 高 |

---

## 3. 验收标准

### 3.1 功能验收

| 验收项 | 验收条件 |
|--------|----------|
| 用户列表 | 登录后可查看所有用户列表 |
| 添加用户 | 点击添加按钮，输入信息后成功创建 |
| 编辑用户 | 点击编辑按钮，修改信息后成功保存 |
| 删除用户 | 点击删除按钮，确认后成功删除 |

---

## 4. 接口需求

| 接口 | 路径 | 方法 | 描述 |
|------|------|------|------|
| 获取用户列表 | /api/users | GET | 获取所有用户 |
| 获取单个用户 | /api/users/{id} | GET | 获取指定用户 |
| 创建用户 | /api/users | POST | 创建新用户 |
| 更新用户 | /api/users/{id} | PUT | 更新用户信息 |
| 删除用户 | /api/users/{id} | DELETE | 删除用户 |

---

## 5. 数据需求

### 5.1 用户信息

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Long | 主键 | 用户ID |
| username | String | 非空，唯一 | 用户名 |
| email | String | 非空，唯一 | 邮箱 |
| password | String | 非空 | 加密密码 |
| createdAt | LocalDateTime | 非空 | 创建时间 |
| updatedAt | LocalDateTime | 非空 | 更新时间 |
