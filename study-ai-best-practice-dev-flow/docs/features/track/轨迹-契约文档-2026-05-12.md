# 轨迹-契约文档

## 1. 接口清单

### 1.1 获取轨迹点列表

| 属性 | 值 |
|------|------|
| 路径 | `GET /api/trackpoints` |
| 方法 | GET |
| 描述 | 获取当前用户的所有轨迹点 |

#### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| userId | Long | 否 | 用户ID（默认当前登录用户） |

#### 成功响应 (200)

```json
[
  {
    "id": 1,
    "userId": 1,
    "latitude": 39.9042,
    "longitude": 116.4074,
    "name": "天安门",
    "description": "首都北京标志性建筑",
    "createdAt": "2024-01-01T10:00:00"
  }
]
```

---

### 1.2 获取单个轨迹点

| 属性 | 值 |
|------|------|
| 路径 | `GET /api/trackpoints/{id}` |
| 方法 | GET |
| 描述 | 获取指定轨迹点 |

#### 路径参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| id | Long | 是 | 轨迹点ID |

#### 成功响应 (200)

```json
{
  "id": 1,
  "userId": 1,
  "latitude": 39.9042,
  "longitude": 116.4074,
  "name": "天安门",
  "description": "首都北京标志性建筑",
  "createdAt": "2024-01-01T10:00:00"
}
```

#### 失败响应 (404)

```json
{
  "status": 404,
  "error": "Not Found",
  "message": "轨迹点不存在"
}
```

---

### 1.3 创建轨迹点

| 属性 | 值 |
|------|------|
| 路径 | `POST /api/trackpoints` |
| 方法 | POST |
| 描述 | 创建新轨迹点 |

#### 请求体

```json
{
  "latitude": "number (必填，纬度)",
  "longitude": "number (必填，经度)",
  "name": "string (可选，位置名称)",
  "description": "string (可选，位置描述)"
}
```

#### 成功响应 (201)

```json
{
  "id": 1,
  "userId": 1,
  "latitude": 39.9042,
  "longitude": 116.4074,
  "name": "天安门",
  "description": "首都北京标志性建筑",
  "createdAt": "2024-01-01T10:00:00"
}
```

#### 失败响应 (400)

```json
{
  "status": 400,
  "error": "Bad Request",
  "message": "纬度和经度不能为空"
}
```

---

### 1.4 更新轨迹点

| 属性 | 值 |
|------|------|
| 路径 | `PUT /api/trackpoints/{id}` |
| 方法 | PUT |
| 描述 | 更新轨迹点信息 |

#### 请求体

```json
{
  "name": "string (可选，位置名称)",
  "description": "string (可选，位置描述)"
}
```

#### 成功响应 (200)

```json
{
  "id": 1,
  "userId": 1,
  "latitude": 39.9042,
  "longitude": 116.4074,
  "name": "更新后的名称",
  "description": "更新后的描述",
  "createdAt": "2024-01-01T10:00:00",
  "updatedAt": "2024-01-01T12:00:00"
}
```

---

### 1.5 删除轨迹点

| 属性 | 值 |
|------|------|
| 路径 | `DELETE /api/trackpoints/{id}` |
| 方法 | DELETE |
| 描述 | 删除轨迹点 |

#### 成功响应 (204)

无响应体

#### 失败响应 (404)

```json
{
  "status": 404,
  "error": "Not Found",
  "message": "轨迹点不存在"
}
```

---

## 2. 数据模型

### 2.1 轨迹点请求模型

| 字段名 | 类型 | 必填 | 约束 | 描述 |
|--------|------|------|------|------|
| latitude | Double | 是 | -90~90 | 纬度 |
| longitude | Double | 是 | -180~180 | 经度 |
| name | String | 否 | 最大100字符 | 位置名称 |
| description | String | 否 | 最大500字符 | 位置描述 |

### 2.2 轨迹点响应模型

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | Long | 轨迹点ID |
| userId | Long | 用户ID |
| latitude | Double | 纬度 |
| longitude | Double | 经度 |
| name | String | 位置名称 |
| description | String | 位置描述 |
| createdAt | LocalDateTime | 创建时间 |
| updatedAt | LocalDateTime | 更新时间 |

---

## 3. 状态码定义

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 204 | 删除成功 |
| 400 | 请求参数错误 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

---

## 4. 业务规则

### 4.1 轨迹点规则

1. 纬度范围：-90° ~ 90°
2. 经度范围：-180° ~ 180°
3. 轨迹点按创建时间顺序排列
4. 轨迹点只属于创建它的用户
5. 用户只能查看和操作自己的轨迹点
