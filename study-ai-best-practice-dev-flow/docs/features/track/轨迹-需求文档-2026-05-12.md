# 轨迹-需求文档

## 1. 需求概述

### 1.1 功能描述

用户历史轨迹功能，基于百度地图实现用户轨迹的记录和展示。

### 1.2 业务背景

用户需要记录自己的行程轨迹，包括在地图上标记位置点，按时间顺序连接成轨迹线路。

### 1.3 需求来源

用户需求："实现用户历史轨迹的一个功能，可以显示用户的历史轨迹图，可基于百度地图实现；可以在地图上，点击位置进行踩点，存储起来，后续进行显示轨迹；轨迹按照时间顺序平滑连接起来"

---

## 2. 功能需求

### 2.1 核心功能

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| FR-010 | 显示百度地图 | 高 |
| FR-011 | 地图上点击踩点 | 高 |
| FR-012 | 保存轨迹点 | 高 |
| FR-013 | 按时间顺序显示轨迹 | 高 |
| FR-014 | 轨迹平滑连接 | 高 |
| FR-015 | 查看历史轨迹列表 | 高 |

### 2.2 非功能需求

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| NFR-007 | 地图加载时间 ≤ 2秒 | 中 |
| NFR-008 | 轨迹显示流畅 | 高 |

---

## 3. 验收标准

### 3.1 功能验收

| 验收项 | 验收条件 |
|--------|----------|
| 地图显示 | 页面加载后百度地图正常显示 |
| 踩点功能 | 点击地图任意位置创建轨迹点 |
| 轨迹保存 | 轨迹点成功保存到数据库 |
| 轨迹展示 | 历史轨迹按时间顺序显示在地图上 |
| 轨迹连接 | 轨迹点之间平滑连接成线 |

### 3.2 性能验收

| 验收项 | 验收条件 |
|--------|----------|
| 地图加载 | 地图加载时间 ≤ 2秒 |
| 轨迹渲染 | 100个轨迹点渲染时间 ≤ 1秒 |

---

## 4. 接口需求

| 接口 | 路径 | 方法 | 描述 |
|------|------|------|------|
| 获取轨迹列表 | /api/trackpoints | GET | 获取当前用户轨迹点列表 |
| 获取单个轨迹点 | /api/trackpoints/{id} | GET | 获取指定轨迹点 |
| 创建轨迹点 | /api/trackpoints | POST | 创建新轨迹点 |
| 更新轨迹点 | /api/trackpoints/{id} | PUT | 更新轨迹点信息 |
| 删除轨迹点 | /api/trackpoints/{id} | DELETE | 删除轨迹点 |

---

## 5. 数据需求

### 5.1 轨迹点信息

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Long | 主键 | 轨迹点ID |
| userId | Long | 非空，外键 | 用户ID |
| latitude | Double | 非空 | 纬度 |
| longitude | Double | 非空 | 经度 |
| name | String | 可选 | 轨迹点名称 |
| description | String | 可选 | 轨迹点描述 |
| createdAt | LocalDateTime | 非空 | 创建时间 |

---

## 6. 技术方案

### 6.1 前端技术

- **地图服务**：百度地图API
- **地图容器**：Vue组件
- **轨迹绘制**：百度地图Polyline

### 6.2 后端技术

- **数据存储**：MySQL/H2
- **数据模型**：TrackPoint实体
