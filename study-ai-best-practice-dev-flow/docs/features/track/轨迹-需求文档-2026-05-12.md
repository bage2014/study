## 一、需求概述

### 1.1 需求来源
- 产品需求：用户需要记录和展示旅行轨迹
- 用户反馈：希望能够在地图上查看自己的足迹

### 1.2 功能描述
实现轨迹点的创建、编辑、删除和展示功能，支持在地图上标记位置，记录旅行足迹，并以时间线形式展示轨迹历史。

### 1.3 业务价值
- 记录用户的旅行经历
- 可视化展示轨迹路线
- 支持分享旅行故事
- 纪念意义和回忆价值

### 1.4 竞对分析

| 竞品 | 优势 | 劣势 | 借鉴点 |
|------|------|------|--------|
| 高德地图足迹 | 地图数据丰富 | 功能单一 | 地图展示 |
| 百度地图 | 定位准确 | 社交功能弱 | 定位服务 |
| 旅行APP | 社交属性强 | 地图功能弱 | 用户体验 |

### 1.5 竞争力分析

**核心优势：**
- 集成百度地图SDK
- 支持精确坐标选择
- 时间线形式展示
- 与家族管理结合

## 三、交互流程

### 3.1 创建轨迹点流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端

    User->>Frontend: 打开轨迹页面
    Frontend->>Frontend: 加载百度地图
    Frontend->>Backend: GET /api/trackpoints
    Backend-->>Frontend: 返回轨迹点列表
    Frontend->>Frontend: 在地图上绘制轨迹点
    
    User->>Frontend: 点击地图位置
    Frontend->>Frontend: 获取点击坐标
    Frontend->>Frontend: 显示位置信息弹窗
    
    User->>Frontend: 输入轨迹点名称和描述
    User->>Frontend: 点击"保存"按钮
    Frontend->>Backend: POST /api/trackpoints
    Backend->>Backend: 保存轨迹点记录
    Backend-->>Frontend: 返回轨迹点信息
    Frontend->>Frontend: 在地图上添加标记
    Frontend->>Frontend: 更新轨迹列表
```

### 3.2 查看轨迹时间线流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端

    User->>Frontend: 点击"时间线"标签
    Frontend->>Backend: GET /api/trackpoints?sort=createdAt
    Backend-->>Frontend: 返回按时间排序的轨迹点列表
    Frontend->>Frontend: 渲染时间线组件
    Frontend->>Frontend: 按时间顺序展示轨迹点
```

### 3.3 绘制轨迹路线流程

```mermaid
sequenceDiagram
    participant User as 用户

    User->>Frontend: 点击"绘制路线"按钮

    Backend-->>Frontend: 返回轨迹点列表
    Frontend->>Frontend: 根据坐标绘制连接线
    Frontend->>Frontend: 更新地图显示
```

## 四、数据流向

```
用户操作 → 前端处理 → API请求 → 后端处理 → 返回响应 → 更新UI
```

### 4.1 创建轨迹点数据流
1. 用户打开轨迹页面
2. 前端加载百度地图
3. 获取已有轨迹点并绘制
4. 用户点击地图位置
5. 显示位置信息弹窗
6. 输入名称和描述
7. 调用POST /api/trackpoints
8. 后端保存轨迹点
9. 返回轨迹点信息
10. 更新地图和列表

### 4.2 查看时间线数据流
1. 用户点击时间线标签
2. 调用GET /api/trackpoints?sort=createdAt
3. 后端返回按时间排序的轨迹点
4. 前端渲染时间线组件

## 六、接口定义

### 6.1 创建轨迹点

**POST /api/trackpoints**

请求体：
```json
{
  "latitude": "number (必填，纬度)",
  "longitude": "number (必填，经度)",
  "name": "string (必填，轨迹点名称)",
  "description": "string (可选，描述)",
  "address": "string (可选，地址)",
  "visitDate": "string (可选，访问日期)"
}
```

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "number",
    "latitude": "number",
    "longitude": "number",
    "name": "string",
    "description": "string",
    "address": "string",
    "visitDate": "string",
    "createdAt": "string"
  }
}
```

### 6.2 获取轨迹点列表

**GET /api/trackpoints**

请求参数：
| 参数 | 类型 | 说明 |
|------|------|------|
| page | number | 页码 |
| size | number | 每页数量 |
| sort | string | 排序字段 |

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [
      {
        "id": "number",
        "latitude": "number",
        "longitude": "number",
        "name": "string",
        "address": "string",
        "createdAt": "string"
      }
    ],
    "totalElements": "number",
    "totalPages": "number"
  }
}
```

### 6.3 更新轨迹点

**PUT /api/trackpoints/{id}**

请求体：
```json
{
  "name": "string (可选)",
  "description": "string (可选)",
  "address": "string (可选)",
  "visitDate": "string (可选)"
}
```

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

### 6.4 删除轨迹点

**DELETE /api/trackpoints/{id}**

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

## 八、错误处理

### 8.1 错误类型

| 错误码 | 错误信息 | 处理方式 |
|--------|----------|----------|
| 400 | 请求参数错误 | 显示表单验证提示 |
| 401 | 未授权 | 跳转到登录页面 |
| 404 | 资源不存在 | 显示资源不存在提示 |
| 500 | 服务器错误 | 显示通用错误提示 |

### 8.2 错误处理流程

```mermaid
flowchart TD
    A[请求失败] --> B{错误码}
    B -->|400| C[显示表单验证错误]
    B -->|401| D[清除登录状态]
    D --> E[跳转到登录页面]
    B -->|404| F[显示资源不存在提示]
    B -->|500| G[显示服务器错误提示]
```
