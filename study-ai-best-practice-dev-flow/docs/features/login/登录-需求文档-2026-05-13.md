## 一、需求概述

### 1.1 需求来源
- 产品需求：用户需要安全的登录机制
- 用户反馈：需要支持注册功能

### 1.2 功能描述
实现用户登录和注册功能，支持用户名/密码认证方式，提供安全、流畅的用户体验。

### 1.3 业务价值
- 保障系统安全性
- 实现用户身份识别
- 支持用户数据隔离
- 提供良好的用户体验

### 1.4 竞对分析

| 竞品 | 优势 | 劣势 | 借鉴点 |
|------|------|------|--------|
| 微信登录 | 一键登录，体验好 | 依赖第三方 | 简化登录流程 |
| 支付宝登录 | 安全级别高 | 流程繁琐 | 安全验证机制 |
| 传统账号密码 | 自主性强 | 记忆负担 | 密码强度提示 |
| 手机验证码 | 便捷 | 依赖短信 | 多因素认证 |

### 1.5 竞争力分析

**核心优势：**
- 简洁的登录界面设计
- 支持账号密码+JWT认证
- 登录态持久化（localStorage）
- 完善的错误提示机制

## 三、交互流程

### 3.1 登录流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端

    User->>Frontend: 打开登录页面
    Frontend->>Frontend: 渲染登录表单
    
    User->>Frontend: 输入用户名/密码
    Frontend->>Frontend: 表单验证（非空检查、长度验证）
    
    User->>Frontend: 点击登录按钮
    Frontend->>Backend: POST /api/users/login
    Backend->>Backend: 验证用户名密码（BCrypt比对）
    alt 验证成功
        Backend-->>Frontend: 返回JWT Token和用户信息
        Frontend->>Frontend: 保存Token到localStorage
        Frontend->>Frontend: 保存用户信息到Pinia store
        Frontend->>Frontend: 跳转到首页(/family)
    else 验证失败
        Backend-->>Frontend: 返回错误信息(401)
        Frontend->>Frontend: 显示错误提示（用户名或密码错误）
    end
```

### 3.2 注册流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端

    User->>Frontend: 切换到注册标签
    Frontend->>Frontend: 渲染注册表单
    
    User->>Frontend: 输入用户名/邮箱/密码
    Frontend->>Frontend: 表单验证（格式、长度、强度）
    
    User->>Frontend: 点击注册按钮
    Frontend->>Backend: POST /api/users/register
    Backend->>Backend: 检查用户名/邮箱是否已存在
    alt 注册成功
        Backend->>Backend: BCrypt加密密码
        Backend->>Backend: 创建用户记录
        Backend-->>Frontend: 返回成功消息
        Frontend->>Frontend: 自动切换到登录页面
        Frontend->>Frontend: 显示注册成功提示
    else 用户名已存在
        Backend-->>Frontend: 返回错误信息(409)
        Frontend->>Frontend: 显示错误提示（用户名已存在）
    else 邮箱已存在
        Backend-->>Frontend: 返回错误信息(409)
        Frontend->>Frontend: 显示错误提示（邮箱已存在）
    end
```

### 3.3 退出流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端

    User->>Frontend: 点击退出按钮
    Frontend->>Backend: POST /api/users/logout
    Backend-->>Frontend: 返回成功消息
    Frontend->>Frontend: 清除localStorage中的Token
    Frontend->>Frontend: 清除Pinia store中的用户信息
    Frontend->>Frontend: 跳转到登录页面
```

### 3.4 登录态持久化流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端
    participant Backend as 后端

    User->>Frontend: 刷新页面
    Frontend->>Frontend: 检查localStorage中是否有Token
    alt Token存在且有效
        Frontend->>Backend: GET /api/users/me (携带Token)
        Backend->>Backend: 验证Token
        Backend-->>Frontend: 返回用户信息
        Frontend->>Frontend: 初始化Pinia store
        Frontend->>Frontend: 渲染首页
    else Token不存在或无效
        Frontend->>Frontend: 渲染登录页面
    end
```

## 四、数据流向

```
用户输入 → 前端验证 → API请求 → 后端验证 → 返回响应 → 更新登录状态
```

### 4.1 登录数据流
1. 用户输入用户名和密码
2. 前端进行基本验证（非空检查、长度验证）
3. 调用登录API（POST /api/users/login）
4. 后端验证用户名密码（BCrypt比对）
5. 返回JWT Token和用户信息
6. 前端保存Token到localStorage
7. 前端保存用户信息到Pinia store
8. 跳转到首页

### 4.2 注册数据流
1. 用户输入用户名、邮箱、密码
2. 前端验证输入格式（邮箱格式、密码强度）
3. 调用注册API（POST /api/users/register）
4. 后端检查用户名/邮箱是否已存在
5. BCrypt加密密码
6. 创建用户记录
7. 返回成功消息
8. 跳转到登录页面

## 六、接口定义

### 6.1 登录接口

**POST /api/users/login**

请求体：
```json
{
  "username": "string (必填，用户名，长度3-64)",
  "password": "string (必填，密码，长度6-128)"
}
```

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "string (JWT令牌)",
    "user": {
      "id": "number (用户ID)",
      "username": "string (用户名)",
      "email": "string (邮箱)"
    }
  }
}
```

失败响应（401）：
```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

### 6.2 注册接口

**POST /api/users/register**

请求体：
```json
{
  "username": "string (必填，用户名，长度3-64)",
  "email": "string (必填，邮箱)",
  "password": "string (必填，密码，长度6-128)"
}
```

成功响应（200）：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": null
}
```

失败响应（409）：
```json
{
  "code": 409,
  "message": "用户名已存在",
  "data": null
}
```

### 6.3 退出接口

**POST /api/users/logout**

成功响应（200）：
```json
{
  "code": 200,
  "message": "退出成功",
  "data": null
}
```

### 6.4 获取当前用户接口

**GET /api/users/me**

请求头：
```
Authorization: Bearer <token>
```

成功响应（200）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "number",
    "username": "string",
    "email": "string",
    "nickname": "string",
    "avatar": "string"
  }
}
```

## 八、错误处理

### 8.1 错误类型

| 错误码 | 错误信息 | 处理方式 |
|--------|----------|----------|
| 400 | 请求参数错误 | 显示表单验证提示 |
| 401 | 未授权 | 跳转到登录页面 |
| 404 | 用户不存在 | 显示友好提示 |
| 409 | 用户名/邮箱已存在 | 显示重复提示 |
| 500 | 服务器错误 | 显示通用错误提示 |

### 8.2 错误处理流程

```mermaid
flowchart TD
    A[请求失败] --> B{错误码}
    B -->|400| C[显示表单验证错误]
    B -->|401| D[清除登录状态]
    D --> E[跳转到登录页面]
    B -->|404| F[显示资源不存在提示]
    B -->|409| G[显示重复提示]
    B -->|500| H[显示服务器错误提示]
```
