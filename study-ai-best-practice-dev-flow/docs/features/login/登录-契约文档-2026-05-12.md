# 登录-契约文档

## 1. 接口清单

### 1.1 用户登录接口

| 属性 | 值 |
|------|------|
| 路径 | `POST /api/users/login` |
| 方法 | POST |
| 描述 | 用户登录验证 |

#### 请求体

```json
{
  "username": "string (必填，用户名)",
  "password": "string (必填，密码)"
}
```

#### 成功响应 (200)

```json
{
  "id": "number (用户ID)",
  "username": "string (用户名)",
  "email": "string (邮箱地址)",
  "createdAt": "string (创建时间，ISO格式)",
  "updatedAt": "string (更新时间，ISO格式)"
}
```

#### 失败响应 (401)

```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "用户名或密码错误"
}
```

#### 失败响应 (400)

```json
{
  "status": 400,
  "error": "Bad Request",
  "message": "用户名和密码不能为空"
}
```

---

## 2. 数据模型

### 2.1 用户登录请求模型

| 字段名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| username | String | 是 | 用户登录名 |
| password | String | 是 | 用户密码 |

### 2.2 用户响应模型

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | Long | 用户唯一标识 |
| username | String | 用户登录名 |
| email | String | 用户邮箱地址 |
| createdAt | LocalDateTime | 创建时间 |
| updatedAt | LocalDateTime | 更新时间 |

---

## 3. 状态码定义

| 状态码 | 含义 |
|--------|------|
| 200 | 登录成功 |
| 400 | 请求参数错误 |
| 401 | 身份验证失败 |
| 500 | 服务器内部错误 |

---

## 4. 业务规则

### 4.1 登录验证规则

1. 用户名和密码不能为空
2. 密码采用BCrypt加密存储，验证时需进行密码比对
3. 登录成功后返回用户信息（不包含密码）
4. 连续失败5次后锁定账户10分钟（可选扩展）
