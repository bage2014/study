# 注册-契约文档

## 1. 接口清单

### 1.1 用户注册接口

| 属性 | 值 |
|------|------|
| 路径 | `POST /api/users/register` |
| 方法 | POST |
| 描述 | 用户注册创建账户 |

#### 请求体

```json
{
  "username": "string (必填，3-20字符)",
  "email": "string (必填，邮箱格式)",
  "password": "string (必填，至少6位)"
}
```

#### 成功响应 (201)

```json
{
  "id": "number (用户ID)",
  "username": "string (用户名)",
  "email": "string (邮箱地址)",
  "createdAt": "string (创建时间，ISO格式)",
  "updatedAt": "string (更新时间，ISO格式)"
}
```

#### 失败响应 (400)

```json
{
  "status": 400,
  "error": "Bad Request",
  "message": "用户名已存在"
}
```

```json
{
  "status": 400,
  "error": "Bad Request",
  "message": "邮箱已存在"
}
```

---

## 2. 数据模型

### 2.1 用户注册请求模型

| 字段名 | 类型 | 必填 | 约束 | 描述 |
|--------|------|------|------|------|
| username | String | 是 | 3-20字符 | 用户登录名 |
| email | String | 是 | 邮箱格式 | 用户邮箱 |
| password | String | 是 | 至少6位 | 用户密码 |

### 2.2 用户响应模型

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | Long | 用户唯一标识 |
| username | String | 用户登录名 |
| email | String | 用户邮箱地址 |
| createdAt | LocalDateTime | 创建时间 |
| updatedAt | LocalDateTime | 更新时间 |

---

## 3. 状态码定义

| 状态码 | 含义 |
|--------|------|
| 201 | 注册成功 |
| 400 | 请求参数错误 |
| 500 | 服务器内部错误 |

---

## 4. 业务规则

### 4.1 注册验证规则

1. 用户名、邮箱、密码不能为空
2. 用户名长度3-20字符
3. 邮箱必须符合邮箱格式
4. 密码至少6位
5. 用户名和邮箱必须唯一
6. 密码使用BCrypt加密存储
