# 注册-需求文档

## 1. 需求概述

### 1.1 功能描述

用户注册功能，允许新用户创建账户，包含用户名、邮箱和密码信息。

### 1.2 业务背景

为了让新用户能够使用系统，需要提供注册功能创建新账户。

### 1.3 需求来源

用户需求："实现用户登陆注册功能"

---

## 2. 功能需求

### 2.1 核心功能

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| FR-007 | 用户输入用户名、邮箱、密码进行注册 | 高 |
| BR-007 | 密码使用BCrypt加密存储 | 高 |

### 2.2 非功能需求

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| NFR-003 | 用户名和邮箱唯一性校验 | 高 |
| NFR-004 | 密码强度要求（至少6位） | 中 |

---

## 3. 验收标准

### 3.1 功能验收

| 验收项 | 验收条件 |
|--------|----------|
| 注册成功 | 输入有效信息，成功创建用户 |
| 用户名重复 | 输入已存在用户名，显示"用户名已存在" |
| 邮箱重复 | 输入已存在邮箱，显示"邮箱已存在" |
| 密码加密 | 数据库中密码为BCrypt加密格式 |

### 3.2 性能验收

| 验收项 | 验收条件 |
|--------|----------|
| 响应时间 | 注册请求响应时间 ≤ 500ms |

---

## 4. 接口需求

### 4.1 注册接口

- **路径**：`POST /api/users/register`
- **请求体**：`{"username": "string", "email": "string", "password": "string"}`
- **成功响应**：`{"id": 1, "username": "test", "email": "test@test.com"}`
- **失败响应**：`{"status": 400, "message": "用户名已存在"}`

---

## 5. 数据需求

### 5.1 用户注册信息

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| username | String | 非空，唯一，3-20字符 | 用户名 |
| email | String | 非空，唯一，邮箱格式 | 邮箱地址 |
| password | String | 非空，至少6位 | 密码 |
