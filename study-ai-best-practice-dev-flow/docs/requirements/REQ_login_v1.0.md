# REQ_login_v1.0 - 用户登录需求文档

## 1. 需求概述

### 1.1 功能描述

用户登录功能，允许已注册用户通过用户名和密码进行身份验证。

### 1.2 业务背景

为保护系统数据安全，只有经过身份验证的用户才能访问系统功能。

### 1.3 需求来源

用户需求："实现用户登陆注册功能"

## 4. 接口需求

### 4.1 登录接口

| 属性 | 值 |
|------|------|
| 路径 | POST /api/users/login |
| 方法 | POST |
| 所属模块 | 用户模块 |

#### 请求体

```json
{
  "username": "string (必填，3-20字符)",
  "password": "string (必填，至少6位)"
}
```

#### 成功响应 (200)

```json
{
  "id": 1,
  "username": "admin",
  "email": "admin@test.com",
  "createdAt": "2026-05-12T10:00:00",
  "updatedAt": "2026-05-12T10:00:00"
}
```

#### 失败响应 (401)

```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "用户名或密码错误"
}
```

## 6. 业务规则

### 6.1 登录验证规则

1. 用户名和密码不能为空
2. 密码采用BCrypt加密存储，验证时进行密码比对
3. 登录成功后返回用户信息（不包含密码）
4. 连续失败5次后锁定账户10分钟（可选扩展）

### 6.2 状态管理规则

1. 登录成功后将用户信息存储在Session或Token中
2. 前端将用户信息存储在localStorage中保持状态
3. 退出登录时清除所有存储的用户信息
