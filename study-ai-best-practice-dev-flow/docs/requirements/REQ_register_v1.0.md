# REQ_register_v1.0 - 用户注册需求文档

## 1. 需求概述

### 1.1 功能描述

用户注册功能，允许新用户创建账户，包含用户名、邮箱和密码信息。

### 1.2 业务背景

为了让新用户能够使用系统，需要提供注册功能创建新账户。

### 1.3 需求来源

用户需求："实现用户登陆注册功能"

|------|------|
| 文档版本 | v1.0 |
| 创建日期 | 2026-05-12 |
| 状态 | 已确认 |

## 2. 功能需求

### 2.1 核心需求

| 需求编号 | 描述 | 优先级 | 来源 |
|----------|------|--------|------|
| FR-007 | 用户输入用户名、邮箱、密码进行注册 | 高 | 用户需求 |
| BR-007 | 密码使用BCrypt加密存储 | 高 | 安全需求 |
| FR-007-01 | 用户名唯一性校验 | 高 | 数据完整性 |
| FR-007-02 | 邮箱唯一性校验 | 高 | 数据完整性 |
| FR-007-03 | 支持从注册弹窗切换到登录页面 | 中 | 用户体验 |

### 2.2 非功能需求

| 需求编号 | 描述 | 优先级 |
|----------|------|--------|
| NFR-004 | 密码强度要求（至少6位） | 中 | 安全需求 |
| NFR-005 | 注册请求响应时间 ≤ 500ms | 中 | 性能需求 |

## 3. 验收标准

### 3.1 功能验收

| 验收项 | 验收条件 | 测试方法 |
|--------|----------|----------|
| 注册成功 | 输入有效信息，成功创建用户 | 手动测试 |
| 用户名重复 | 输入已存在用户名，显示"用户名已存在" | 手动测试 |
| 邮箱重复 | 输入已存在邮箱，显示"邮箱已存在" | 手动测试 |
| 密码加密 | 数据库中密码为BCrypt加密格式 | 代码审查 |
| 切换登录 | 点击"立即登录"链接，切换到登录弹窗 | 手动测试 |

## 4. 接口需求

### 4.1 注册接口

| 属性 | 值 |
|------|------|
| 路径 | POST /api/users/register |
| 方法 | POST |
| 所属模块 | 用户模块 |

#### 请求体

```json
{
  "username": "string (必填，3-20字符)",
  "email": "string (必填，邮箱格式)",
  "password": "string (必填，至少6位)"
}
```

#### 成功响应 (201)

```json
{
  "id": 1,
  "username": "newuser",
  "email": "newuser@test.com",
  "createdAt": "2026-05-12T10:00:00",
  "updatedAt": "2026-05-12T10:00:00"
}
```

#### 失败响应 (400)

```json
{
  "status": 400,
  "error": "Bad Request",
  "message": "用户名已存在"
}
```

## 5. 数据需求

### 5.1 用户实体

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Long | 主键，自增 | 用户ID |
| username | String | 非空，唯一，3-20字符 | 用户名 |
| email | String | 非空，唯一，邮箱格式 | 邮箱地址 |
| password | String | 非空，BCrypt加密 | 密码 |
| createdAt | LocalDateTime | 非空 | 创建时间 |
| updatedAt | LocalDateTime | 非空 | 更新时间 |

---

## 6. 业务规则

### 6.1 注册验证规则

1. 用户名、邮箱、密码不能为空
2. 用户名长度3-20字符
3. 邮箱必须符合邮箱格式
4. 密码至少6位
5. 用户名和邮箱必须唯一
6. 密码使用BCrypt加密存储
