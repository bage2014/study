# REQ_track_v1.0 - 用户轨迹需求文档

## 1. 需求概述

### 1.1 功能描述

用户历史轨迹功能，基于百度地图实现用户轨迹的记录和展示。

### 1.2 业务背景

用户需要记录自己的行程轨迹，包括在地图上标记位置点，按时间顺序连接成轨迹线路。

### 1.3 需求来源

用户需求："实现用户历史轨迹的一个功能，可以显示用户的历史轨迹图，可基于百度地图实现；可以在地图上，点击位置进行踩点，存储起来，后续进行显示轨迹；轨迹按照时间顺序平滑连接起来"

## 4. 接口需求

### 4.1 接口列表

| 接口 | 路径 | 方法 | 描述 |
|------|------|------|------|
| 获取轨迹列表 | GET /api/trackpoints | GET | 获取当前用户轨迹点列表 |
| 获取单个轨迹点 | GET /api/trackpoints/{id} | GET | 获取指定轨迹点 |
| 创建轨迹点 | POST /api/trackpoints | POST | 创建新轨迹点 |
| 更新轨迹点 | PUT /api/trackpoints/{id} | PUT | 更新轨迹点信息 |
| 删除轨迹点 | DELETE /api/trackpoints/{id} | DELETE | 删除轨迹点 |

### 4.2 接口详情

#### POST /api/trackpoints

**请求体**：

```json
{
  "latitude": "number (必填，-90~90)",
  "longitude": "number (必填，-180~180)",
  "name": "string (可选，位置名称)",
  "description": "string (可选，位置描述)"
}
```

**成功响应 (201)**：

```json
{
  "id": 1,
  "userId": 1,
  "latitude": 39.9042,
  "longitude": 116.4074,
  "name": "天安门",
  "description": "首都北京标志性建筑",
  "createdAt": "2026-05-12T10:00:00"
}
```

## 5. 数据需求

### 5.1 轨迹点实体

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Long | 主键，自增 | 轨迹点ID |
| userId | Long | 非空，外键 | 用户ID |
| latitude | Double | 非空，-90~90 | 纬度 |
| longitude | Double | 非空，-180~180 | 经度 |
| name | String | 可选，最大100字符 | 位置名称 |
| description | String | 可选，最大500字符 | 位置描述 |
| createdAt | LocalDateTime | 非空 | 创建时间 |
| updatedAt | LocalDateTime | 非空 | 更新时间 |

---

## 6. 业务规则

### 6.1 轨迹点规则

1. 纬度范围：-90° ~ 90°
2. 经度范围：-180° ~ 180°
3. 轨迹点按创建时间顺序排列
4. 轨迹点只属于创建它的用户
5. 用户只能查看和操作自己的轨迹点
