# REQ_user_v1.0 - 用户管理需求文档

## 1. 需求概述

### 1.1 功能描述

用户管理功能，允许管理员查看、添加、编辑和删除用户信息。

### 1.2 业务背景

系统需要对用户进行统一管理，包括查看用户列表、创建新用户、修改用户信息和删除用户。

### 1.3 需求来源

用户需求："登陆后，可以查看当前查看用户列表，实现用户的增删改查功能"

## 4. 接口需求

### 4.1 接口列表

| 接口 | 路径 | 方法 | 描述 |
|------|------|------|------|
| 获取用户列表 | GET /api/users | GET | 获取所有用户 |
| 获取单个用户 | GET /api/users/{id} | GET | 获取指定用户 |
| 创建用户 | POST /api/users | POST | 创建新用户 |
| 更新用户 | PUT /api/users/{id} | PUT | 更新用户信息 |
| 删除用户 | DELETE /api/users/{id} | DELETE | 删除用户 |

### 4.2 接口详情

#### GET /api/users

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认0 |
| size | int | 否 | 每页数量，默认10 |

**响应**：

```json
{
  "content": [
    {
      "id": 1,
      "username": "admin",
      "email": "admin@test.com",
      "createdAt": "2026-05-12T10:00:00"
    }
  ],
  "totalElements": 10,
  "totalPages": 1,
  "number": 0,
  "size": 10
}
```

## 5. 数据需求

### 5.1 用户实体

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Long | 主键，自增 | 用户ID |
| username | String | 非空，唯一，3-20字符 | 用户名 |
| email | String | 非空，唯一，邮箱格式 | 邮箱地址 |
| password | String | 非空，BCrypt加密 | 密码 |
| createdAt | LocalDateTime | 非空 | 创建时间 |
| updatedAt | LocalDateTime | 非空 | 更新时间 |

---

## 6. 业务规则

### 6.1 用户管理规则

1. 只有登录用户才能访问用户管理页面
2. 删除用户前必须确认
3. 不允许删除当前登录用户
4. 用户密码在列表中不显示
