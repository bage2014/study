# /start 命令
# 需求澄清与启动命令

## 功能描述
进行需求澄清并生成需求文档。

## 自动执行的任务序列

1. **需求收集**
   - 与用户进行头脑风暴
   - 明确功能范围
   - 确定优先级

2. **需求细化**
   - 将需求分解为可执行的功能点
   - 定义输入输出
   - 确定边界条件
   - 识别依赖关系

3. **生成需求文档**
   - 创建 `docs/requirements/REQ_[模块]_v[版本].md`
   - 包含功能需求、非功能需求、验收标准

4. **拆解子任务**
   - 创建 TODO 列表
   - 分配到后端和前端任务

5. **更新进度**
   - 创建 `docs/reports/DAILY_[日期].md`
   - 记录当前状态和待办事项

## 使用方式

```
/start [需求描述]
```

## 示例

```
/start 实现用户登录注册功能
/start 实现用户历史轨迹功能
```

## 输出

- 需求文档：`docs/requirements/REQ_[模块]_v[版本].md`
- 每日进度：`docs/reports/DAILY_[日期].md`
- 功能模块文档：`docs/features/[功能]/`

## 命名规范

需求文档命名：`REQ_[功能简写]_v[版本].md`
- 例如：`REQ_login_v1.0.md`、`REQ_track_v1.0.md`
