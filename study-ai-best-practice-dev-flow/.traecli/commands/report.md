# /report 命令
# 进度报告生成命令

## 功能描述
生成项目进度报告和测试报告。

## 自动执行的任务序列

1. **需求状态汇总**
   - 列出所有需求及其状态
   - 更新需求文档进度

2. **代码状态汇总**
   - 统计代码变更
   - 检查代码质量

3. **测试状态汇总**
   - 汇总单元测试结果
   - 汇总 Playwright 测试结果
   - 生成测试覆盖率报告

4. **生成报告**
   - 生成每日进度报告 `docs/reports/DAILY_[日期].md`
   - 生成测试报告 `docs/reports/TEST_[日期].md`
   - 更新需求文档状态

5. **风险评估**
   - 识别当前风险
   - 提出解决建议

## 使用方式

```
/report
/report [类型]
```

## 示例

```
/report
/report daily
/report test
```

## 输出

### 每日进度报告 (DAILY)
- 今日完成内容
- 明日计划
- 遇到的问题
- 风险评估

### 测试报告 (TEST)
- 测试执行概况
- 通过率统计
- 覆盖率统计
- 失败用例分析

## 报告格式

报告存放在 `docs/reports/` 目录：
- `DAILY_2026-05-12.md` - 每日进度
- `TEST_2026-05-12.md` - 测试报告
