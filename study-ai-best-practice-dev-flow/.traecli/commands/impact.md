# /impact 命令
# 变更影响分析命令

## 功能描述
分析代码变更的影响范围，评估需要同步修改的内容。

## 自动执行的任务序列

1. **文件引用分析**
   - 分析被修改文件的引用关系
   - 找出依赖该文件的其他模块

2. **影响范围评估**
   - 评估对后端 API 的影响
   - 评估对前端组件的影响
   - 评估对数据库的影响

3. **测试影响分析**
   - 列出需要更新的单元测试
   - 列出需要更新的 Playwright 测试

4. **文档影响分析**
   - 列出需要更新的文档

5. **输出影响报告**
   - 生成变更影响分析报告

## 使用方式

```
/impact [变更描述]
```

## 示例

```
/impact 修改用户登录接口的响应格式
/impact 删除用户管理模块的某个API
```

## 输出

影响分析报告，包含：
- 受影响的文件列表
- 需要同步修改的测试
- 需要同步修改的文档
- 风险评估

## 风险等级

| 等级 | 说明 |
|------|------|
| 高 | 可能导致系统不可用或数据丢失 |
| 中 | 可能导致功能异常或性能下降 |
| 低 | 基本无影响，仅影响局部功能 |
