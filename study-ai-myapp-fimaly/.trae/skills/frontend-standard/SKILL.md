# 前端开发标准规范

## 1. 页面布局结构

### 1.1 基本结构
所有前端页面应遵循以下基本结构：

```vue
<template>
  <div class="min-h-screen bg-gray-50">
    <Header title="页面标题" />
    <main class="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-8">
      <!-- 页面内容 -->
    </main>
  </div>
</template>

<script setup>
// 导入组件和store
import Header from '../components/Header.vue'
// 其他导入和逻辑
</script>
```

### 1.2 头部组件
使用统一的 `Header` 组件，包含以下功能：
- 返回按钮：点击返回首页
- 页面标题：显示当前页面名称
- 操作按钮：通过插槽 `actions` 添加页面特定操作
- 用户信息：显示当前登录用户
- 退出登录按钮：点击退出登录

## 2. 组件使用规范

### 2.1 通用组件
- **Header**：页面头部，包含返回按钮、页面标题、操作按钮和用户信息
- **Modal**：弹窗组件，用于表单提交、确认操作等
- **Form**：表单组件，包含输入框、下拉选择、日期选择等
- **Table**：表格组件，用于展示数据列表
- **Card**：卡片组件，用于组织相关内容

### 2.2 状态管理
使用 Pinia 进行状态管理，遵循以下规范：
- 每个功能模块创建独立的 store
- 命名规范：`use{ModuleName}Store`
- 包含 state、getters、actions
- 异步操作使用 async/await

## 3. 样式规范

### 3.1 布局样式
- 使用 Tailwind CSS 进行样式管理
- 容器宽度：`max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- 间距规范：使用 Tailwind 的间距类，如 `p-4`、`m-4`、`space-x-4` 等
- 卡片样式：`bg-white p-6 rounded-lg shadow`

### 3.2 颜色规范
- 主色调：绿色系列（`green-500`、`green-600`）
- 辅助色：蓝色系列（`blue-500`）、紫色系列（`purple-600`）
- 文字颜色：`text-gray-900`（标题）、`text-gray-700`（正文）、`text-gray-500`（辅助文字）
- 背景颜色：`bg-gray-50`（页面背景）、`bg-white`（卡片背景）

### 3.3 响应式设计
- 使用 Tailwind 的响应式类，如 `md:grid-cols-2`、`lg:grid-cols-3`
- 确保在不同屏幕尺寸下的良好显示

## 4. 代码规范

### 4.1 命名规范
- 组件名称：PascalCase（如 `Header.vue`）
- 变量名称：camelCase（如 `userStore`）
- 常量名称：SCREAMING_SNAKE_CASE（如 `API_BASE_URL`）
- 函数名称：camelCase（如 `fetchData`）
- 文件名称：组件文件使用 PascalCase，其他文件使用 kebab-case

### 4.2 代码结构
- 使用 `<script setup>` 语法
- 按顺序导入：Vue 核心 API、第三方库、本地组件、本地 store
- 组件逻辑按功能分组：状态定义、计算属性、生命周期钩子、事件处理函数

### 4.3 注释规范
- 关键逻辑添加注释
- 复杂函数添加函数说明
- 组件参数添加类型注释
- 使用 JSDoc 格式进行函数和组件文档

### 4.4 代码风格
- 使用 ES6+ 语法
- 代码缩进使用 2 个空格
- 单行代码长度不超过 100 字符
- 适当使用空行分隔逻辑块

## 5. 最佳实践

### 5.1 性能优化
- 使用 `computed` 缓存计算结果
- 使用 `watch` 监听响应式数据变化
- 避免在模板中直接使用复杂表达式
- 合理使用 `v-if` 和 `v-show`
- 组件懒加载和路由懒加载

### 5.2 用户体验
- 添加加载状态（如 `loading` 动画）
- 添加错误处理和错误提示
- 表单验证和反馈
- 响应式设计，适配不同设备
- 合理的动画和过渡效果

### 5.3 安全规范
- 防止 XSS 攻击，使用 `v-html` 时谨慎
- 防止 CSRF 攻击，使用 CSRF token
- 敏感数据加密传输
- 权限控制，确保用户只能访问有权限的资源

- 功能：选择成员、查看事件列表、添加/编辑事件
- 结构：Header + 成员选择器 + 事件列表 + 操作按钮

- 标题：家族地理位置
- 功能：选择家族、查看位置列表、添加/编辑位置、地图显示
- 结构：Header + 家族选择器 + 位置列表 + 地图组件

- 标题：AI关系分析
- 功能：选择家族、分析关系、查看分析结果
- 结构：Header + 家族选择器 + 分析按钮 + 分析结果展示

### 6.4 操作日志页面
- 标题：操作日志
- 功能：查看操作记录、筛选日志

## 7. 维护指南

### 7.1 代码审查
- 定期进行代码审查，确保代码质量
- 检查是否遵循编码规范
- 检查性能问题和安全漏洞

### 7.2 文档维护
- 保持文档更新，反映代码变更
- 记录重要的设计决策和实现细节
- 提供使用示例和最佳实践

### 7.3 版本控制
- 使用 Git 进行版本控制
- 提交信息清晰明了，遵循约定式提交规范
- 定期进行分支合并和版本发布

## 8. 工具和依赖

### 8.1 核心依赖
- Vue 3：前端框架
- Pinia：状态管理
- Vue Router：路由管理
- Tailwind CSS：样式管理
- Axios：HTTP 客户端

### 8.2 开发工具
- Vite：构建工具
- ESLint：代码质量检查
- Prettier：代码格式化
- Vitest：单元测试
- Playwright：端到端测试

### 8.3 第三方服务
- 高德地图 API：地理位置服务
- AI 服务：用于关系分析和故事生成

通过遵循以上标准，可以确保前端页面的一致性、可维护性和用户体验，同时提高开发效率和代码质量。
