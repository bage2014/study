# 家庭族谱APP产品需求文档

## 1. 产品概述

### 1.1 产品简介

家庭族谱APP是一款用于记录、管理和展示家族成员信息的应用，支持Web端和Android端。通过该应用，用户可以创建完整的家族树，记录家族历史和重要事件，上传和管理家族多媒体资料，实现家族文化的传承和分享。

### 1.2 产品目标

- 帮助用户便捷地记录和管理家族成员信息
- 提供直观的家族树可视化展示
- 支持家族历史和重要事件的记录
- 实现家族多媒体资料的管理和分享
- 保障用户数据安全和隐私

### 1.3 目标用户

- 普通家庭用户：希望记录和管理家族成员信息，了解家族历史
- 家谱研究者：需要专业的工具来整理和分析家族数据
- 老年人：希望为后代留下家族历史记录
- 年轻人：对家族历史感兴趣，希望通过现代化工具了解家族背景

### 1.4 竞品分析

#### 1.4.1 主要竞品

| 竞品名称 | 优势 | 劣势 |
|---------|------|------|
| Ancestry.com | 数据丰富，功能强大，全球用户基础 | 价格高，主要面向国外用户，界面复杂 |
| MyHeritage | 界面友好，功能全面，DNA测试集成 | 部分高级功能需要付费，国内访问速度慢 |
| 家谱通 | 符合国内用户习惯，操作简单 | 功能单一，界面设计老旧，缺乏现代化特性 |
| 家族圈 | 社交功能丰富，互动性强 | 家族树功能简单，数据安全性有待提高 |

#### 1.4.3 业界缺失的核心功能

1. **跨代沟通平台**：为不同年龄层的家族成员提供适合的沟通方式
2. **家族健康档案**：记录家族健康历史，提供健康风险评估
3. **简化操作流程**：针对老年人优化的简化操作界面
4. **本地数据备份**：支持离线使用和本地数据备份，确保数据安全
5. **家族文化传承**：提供家族文化、传统习俗的记录和传承功能

**注**：智能关系补全和家族故事生成功能已在本产品中实现。

## 3. 核心流程

### 3.1 用户注册流程

1. 用户打开应用，点击注册按钮
2. 输入手机号/邮箱、密码、验证码
3. 点击注册，系统验证信息
4. 注册成功，自动登录进入首页

### 3.2 家族管理流程

#### 3.2.1 新建家族流程
1. 用户登录后，点击创建家族
2. 输入家族名称、家族描述
3. 上传家族头像（可选）
4. 点击创建，系统生成家族空间
5. 进入家族管理页，开始添加成员

#### 3.2.2 退出家族流程
1. 进入家族管理页
2. 点击退出家族按钮
3. 系统提示确认
4. 确认后，用户退出家族
5. 返回到首页

#### 3.2.3 编辑家族流程
1. 进入成员管理页，选择家族
2. 点击查看/编辑家族按钮
3. 在弹出的模态框中修改家族信息
4. 点击保存，家族信息更新成功

#### 3.2.4 查看家族流程
1. 进入成员管理页，选择家族
2. 查看家族信息卡片
3. 显示家族名称、创建时间、描述等信息

### 3.3 成员管理流程

#### 3.3.1 查看家族成员流程
1. 进入成员管理页，选择家族
2. 系统显示该家族的成员列表
3. 支持按成员信息进行排序和筛选

#### 3.3.2 新建成员流程
1. 进入成员管理页，选择家族
2. 点击添加成员按钮
3. 系统验证是否已选择家族，若未选择则提示用户
4. 输入成员基本信息（姓名、性别、出生日期、去世日期、手机号、邮箱、详细信息等）
5. 上传成员照片（可选）
6. 点击保存，成员添加成功
7. 成员列表自动更新
8. 家族树自动更新

#### 3.3.3 备注成员流程
1. 进入成员详情页
2. 编辑成员的详细信息字段
3. 添加备注信息
4. 点击保存，备注信息更新成功

#### 3.3.4 绑定成员关系流程
1. 进入关系管理页
2. 选择成员1和成员2
3. 支持通过搜索页面选择成员
4. 选择关系类型
5. 点击保存，关系绑定成功
6. 家族树自动更新

### 3.4 家族管理页面流程

1. 进入家族管理页
2. 查看家族列表，选择一个家族
3. 查看当前家族的所有成员和关联关系
4. 点击右上角"新建家族"按钮，创建新家族
5. 点击右上角"查看明细"按钮，查看和编辑当前家族
6. 在当前家族下，点击"添加成员"按钮，添加新成员
7. 在当前家族下，点击"添加关系"按钮，维护成员关联关系
8. 支持删除成员和删除关系
9. 查看家族成员列表和关系列表

### 3.5 家族树页面流程

1. 进入家族树页
2. 查看家族列表，选择一个家族
3. 查看家族关系图显示，直观展示家族成员之间的关系
4. 支持拖拽节点调整布局
5. 查看关系类型标签，了解成员之间的关系

### 3.6 成员搜索流程

1. 进入成员搜索页
2. 输入手机号或邮箱进行搜索
3. 查看搜索结果列表
4. 点击"选择"按钮选择成员
5. 系统将选择的成员信息带回原页面

### 3.7 关系管理流程

1. 进入关系管理页
2. 选择成员1和成员2
3. 支持通过搜索页面选择成员
4. 选择关系类型
5. 点击保存，关系添加成功
6. 家族树自动更新

## 5. 技术实现

### 5.1 技术栈

- **后端**：Spring Boot 3.1+
- **数据库**：H2 in-memory (开发环境)，MySQL 8.0+ (生产环境)
- **Web前端**：Vue 3+、Pinia、Tailwind CSS
- **移动端**：Flutter
- **云存储**：对象存储服务（如OSS）
- **认证**：JWT

### 5.2 架构设计

- **后端**：RESTful API架构，分层设计（Controller、Service、Repository）
- **前端**：单页应用（SPA），组件化设计
- **数据同步**：前后端分离，通过API交互
- **数据存储**：结构化数据存储在MySQL，多媒体文件存储在对象存储服务

### 5.3 API设计规范

- **参数传递**：
  - GET请求：使用查询参数（query parameters）传递参数
  - POST/PUT请求：使用请求体（request body）传递参数
  - **禁止**：将参数直接放在URL路径中，例如 `/families/{familyId}/members` 这种形式是不允许的
  - **正确示例**：
    - GET请求：`/members/family?familyId=1`
    - POST请求：`/members` 配合请求体 `{ "familyId": 1, "name": "张三" }`

### 5.4 安全设计

- **用户认证**：JWT token认证
- **权限管理**：基于角色的访问控制（RBAC）
- **路由拦截**：未登录用户或权限不足（403错误）时自动跳转到登录页面
- **数据加密**：敏感数据加密存储
- **传输安全**：HTTPS协议
- **防止SQL注入**：使用参数化查询
- **防止XSS攻击**：输入验证和输出编码

## 6. 数据需求

### 6.1 核心数据模型

1. **用户表**：用户ID、手机号/邮箱、密码、昵称、头像
2. **家族表**：家族ID、家族名称、描述、创建时间、创建者ID
3. **成员表**：成员ID、家族ID、姓名、性别、出生日期、去世日期、照片、详细信息、手机号、邮箱
4. **关系表**：关系ID、成员ID1、成员ID2、关系类型
5. **历史事件表**：事件ID、家族ID、事件名称、描述、发生时间、相关成员、照片
